# Licensing for community recipe files

Recipe JSON in [`recipes/`](./recipes/) is meant to be **shared** and, in the future, **listed inside the app** via `community-recipes-index.json`.

## Default rule (unless you say otherwise)

By submitting a recipe file in a **pull request** to this repository, you agree that the file may be distributed with the community hub and the app under **Creative Commons Attribution 4.0** (**CC BY 4.0**): others may share and adapt it with attribution.

State in your PR if you need a **different** license (for example “all rights reserved—display only, no derivatives”) so maintainers can mark the manifest entry accordingly.

## What to put in the recipe

The app’s export format may not include a license field yet. Until it does, include in the PR description:

- **Your name or handle** (attribution)
- **License choice** (default CC BY 4.0 if omitted)
- **Optional:** bean, batch size, machine, and anything reviewers should know

## Not legal advice

This document is a practical policy for a small open hub—not a law firm memo. Adjust text with a lawyer if the project grows materially.
